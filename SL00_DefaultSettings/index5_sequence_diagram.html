비행 스케줄 페이지 (index5.jsp) - 시퀀스 다이어그램

페이지 개요: 실시간 비행 스케줄 현황을 조회하고 관리하는 관리자 페이지입니다.

주요 기능: 날짜별 스케줄 조회, 스케줄 DB 저장, 최신 데이터 동기화, 페이지네이션

1. 페이지 초기 로딩 및 초기 데이터 표시

sequenceDiagram participant User as 사용자 participant Browser as 브라우저 participant JSP as index5.jsp participant ScheduleCtrl as FlightScheduleController participant Service as FlightScheduleService participant JS as JavaScript User->>Browser: /views/index5 접속 Browser->>ScheduleCtrl: GET /views/index5 ScheduleCtrl->>Service: 오늘 날짜 스케줄 조회 Service->>ScheduleCtrl: 초기 데이터 반환 ScheduleCtrl->>JSP: return "adminpage/index5" + initialData JSP->>Browser: HTML 페이지 렌더링 (initialData 포함) Browser->>JS: DOMContentLoaded 이벤트 alt 초기 데이터 존재 JS->>JS: currentSchedules = initialData JS->>Browser: 테이블 및 페이지네이션 렌더링 else 초기 데이터 없음 JS->>JS: loadAndDisplaySchedules(오늘날짜, 'all') end

2. 날짜 선택 및 스케줄 조회

sequenceDiagram participant User as 사용자 participant DatePicker as DateRangePicker participant JS as JavaScript participant ApiCtrl as FlightApiController participant Service as FlightScheduleService User->>DatePicker: 달력에서 날짜 선택 DatePicker->>JS: apply.daterangepicker 이벤트 JS->>JS: selectedDate 업데이트 JS->>JS: loadAndDisplaySchedules(선택날짜, 'all') JS->>JS: showLoading() - 로딩 표시 JS->>ApiCtrl: GET /api/flights?date={날짜}&flightType=all ApiCtrl->>Service: 해당 날짜 스케줄 조회 Service->>Service: 외부 API 또는 DB에서 데이터 조회 Service->>ApiCtrl: 스케줄 데이터 리스트 반환 ApiCtrl->>JS: JSON 응답 (스케줄 배열) alt 조회 성공 JS->>JS: currentSchedules 업데이트 JS->>JS: renderTable() - 테이블 렌더링 JS->>JS: renderPagination() - 페이지네이션 생성 else 조회 실패 JS->>Browser: "스케줄 조회 실패" 메시지 표시 end JS->>JS: hideLoading() - 로딩 숨김

3. 페이지네이션 처리

sequenceDiagram participant User as 사용자 participant JS as JavaScript participant DOM as DOM Element User->>DOM: 페이지 번호 클릭 DOM->>JS: 페이지네이션 클릭 이벤트 JS->>JS: currentPage 업데이트 JS->>JS: renderTable() 호출 JS->>JS: 현재 페이지 데이터만 계산 Note over JS: startIndex = (currentPage-1) * ITEMS_PER_PAGE
endIndex = startIndex + ITEMS_PER_PAGE JS->>DOM: 해당 페이지 스케줄만 테이블에 표시 JS->>JS: renderPagination() - 페이지 버튼 상태 업데이트 JS->>Browser: 테이블 영역으로 스크롤 이동

4. 스케줄 데이터 DB 저장

sequenceDiagram participant User as 사용자 participant JS as JavaScript participant SaveCtrl as FlightSaveController participant Service as FlightSaveService User->>JS: "DB에 저장하기" 버튼 클릭 JS->>JS: 현재 스케줄 데이터 유효성 검사 alt 저장할 데이터 없음 JS->>User: "저장할 스케줄 데이터가 없습니다" 알림 else 데이터 존재 JS->>JS: 버튼 비활성화 및 로딩 상태 JS->>SaveCtrl: POST /api/flights/save Note over JS,SaveCtrl: JSON: {
"schedules": currentSchedules,
"date": selectedDate,
"filterType": "all",
"timestamp": ISO날짜
} SaveCtrl->>Service: 스케줄 데이터 DB 저장 처리 Service->>Service: 중복 검사 및 저장 로직 alt 저장 성공 Service->>SaveCtrl: 저장 완료 SaveCtrl->>JS: {"status": "success"} JS->>User: "스케줄 데이터가 성공적으로 저장되었습니다" 알림 else 중복 데이터 Service->>SaveCtrl: 중복 감지 SaveCtrl->>JS: {"status": "Duplicationfail", "message": "중복"} JS->>User: "중복으로 실패했습니다" 알림 else 저장 실패 Service->>SaveCtrl: 저장 실패 SaveCtrl->>JS: {"status": "fail", "message": "오류메시지"} JS->>User: "저장 실패" 알림 end JS->>JS: 버튼 상태 복원 end

5. 최신 데이터 동기화

sequenceDiagram participant User as 사용자 participant JS as JavaScript participant ApiCtrl as FlightApiController participant RefreshCtrl as FlightRefreshController participant Service as FlightService User->>JS: "최신데이터 동기화" 버튼 클릭 JS->>JS: 버튼 비활성화 및 로딩 상태 Note over JS: 1단계: 현재 데이터 가져오기 JS->>ApiCtrl: GET /api/flights?date={날짜}&flightType=all ApiCtrl->>JS: 현재 데이터 반환 Note over JS: 2단계: 새로운 데이터 확인 및 추가 JS->>RefreshCtrl: POST /api/flights/refresh Note over JS,RefreshCtrl: JSON: {
"currentSchedules": currentData,
"date": selectedDate,
"filterType": "all"
} RefreshCtrl->>Service: 외부 API에서 최신 스케줄 조회 Service->>Service: 기존 데이터와 비교하여 새로운 스케줄 식별 alt 새로운 스케줄 있음 Service->>RefreshCtrl: 새로운 스케줄 데이터 RefreshCtrl->>JS: {"status": "success", "newSchedules": [...], "message": "N개 추가"} JS->>JS: currentSchedules에 새 데이터 추가 JS->>JS: renderTable() 및 renderPagination() 재실행 JS->>User: "새로운 스케줄이 추가되었습니다" 알림 else 새로운 스케줄 없음 Service->>RefreshCtrl: 신규 데이터 없음 RefreshCtrl->>JS: {"status": "no_new_data"} JS->>User: "새로 추가된 스케줄이 없습니다" 알림 else 동기화 실패 Service->>RefreshCtrl: 오류 발생 RefreshCtrl->>JS: {"status": "error", "message": "오류메시지"} JS->>User: "동기화 중 오류가 발생했습니다" 알림 end JS->>JS: 버튼 상태 복원
